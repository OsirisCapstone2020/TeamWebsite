Project Requirements

  1. Integration of a pipeline framework: In order to allow users to quickly and easily create image processing pipelines, we want to integrate a UI framework for building directed, acyclic graphs. We've chosen n8n to integrate into our project for this purpose.
  2. User Input and Output: Users need to specify one or more input images. Images will come from Pilot, the Astrogeology Science Center's library of planetary images. Users also need to be able to specify where output images will be stored after processing. Output options could include receiving the images via email or a USGS-only storage area.
  3. ISIS Processing Server: Images from satellites can be much larger than the average image file, and commands from the ISIS suite can take up a lot of processing power. For this reason, Team Osiris needs to develop a dedicated backend server that will run ISIS commands based on incoming requests from our frontend. The server will run the ISIS command requested by the n8n node and return the result back to that n8n node.
  4. Image Storage: Both the inputs and outputs of the ASC Processing Pipeline are larger-than-normal image files. To deal with this, we need dedicated storage. Team Osiris will integrate Amazon's S3 into the application, decoupling storage from both the UI and processing server. The UI will push pipeline input to the S3 server, where the processing server will pull it down. After a processing command is complete, the processing server will push its output back to S3, where the UI can pick it up again.
  5. Security: Since the ASC Processing Pipeline is essentially a remote command execution server, we need to make sure that only ISIS commands are run, and that only image files are provided as input. This type of validation will be done both in the UI and on the backend.
